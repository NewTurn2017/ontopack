# M6 topic map validation

## Claim

OntoPack now produces a deterministic, read-only tag-based topic map for knowledge maintenance and review.

## Scope

- `pack_core::Pack::topic_map(min_count)` scans source-of-truth markdown notes and builds topic nodes from tags.
- Topic edges are tag co-occurrences within the same note, with supporting note ids retained for citation/debugging.
- `pack topics --min-count N` prints a Korean human-readable topic map; `pack topics --json` emits stable JSON for agents and scripts.
- `scripts/real-test.sh` asserts that the realistic pack exposes the shared `ontology` topic in text and JSON reports.

## Evidence

- `cargo test -p pack-core topic_map` — passed.
- `cargo test -p pack-cli topics_reports_tag_topic_map` — passed.
- `scripts/real-test.sh` — passed with topic-map assertions on a realistic pack.
- `cargo test` — passed.
- `cargo clippy --all-targets -- -D warnings` — passed.
- `cargo check -p pack-cli --features real-embed` — passed.
- `git diff --check` — passed.

## Notes

This slice intentionally uses explicit tags, not embedding-based clustering. Fuzzy clusters, auto-topic naming, and proactive related-note recommendations remain future M6 work.
